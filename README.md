## 1. Termos Gerais de Programação

| Ordem | Termo                   | Descrição                                                                 |
|-------|-------------------------|---------------------------------------------------------------------------|
| 1     | Algoritmo               | Sequência de passos ou instruções para resolver um problema.               |
| 2     | API (Application Programming Interface) | Conjunto de rotinas e padrões para integração entre diferentes sistemas de software. |
| 3     | Array                   | Estrutura de dados que armazena múltiplos valores em uma única variável.   |
| 4     | Bug                     | Erro ou defeito em um programa de software que impede seu funcionamento correto. |
| 5     | Compilador              | Programa que traduz código de uma linguagem de programação para linguagem de máquina. |
| 6     | Depuração (Debugging)   | Processo de encontrar e corrigir erros no código.                           |
| 7     | Framework               | Estrutura de suporte reutilizável para o desenvolvimento de software.      |
| 8     | IDE (Integrated Development Environment) | Ambiente de desenvolvimento integrado que inclui ferramentas para escrever, testar e depurar código. |
| 9     | Iteração                | Repetição de um conjunto de instruções em um loop.                          |
| 10    | Linguagem de Programação | Conjunto de regras e sintaxes usadas para escrever programas de software.   |

## 2. Estruturas de Dados

| Ordem | Termo                   | Descrição                                                                 |
|-------|-------------------------|---------------------------------------------------------------------------|
| 1     | Árvore (Tree)           | Estrutura de dados hierárquica composta por nós conectados.                |
| 2     | Pilha (Stack)           | Estrutura de dados que segue o princípio LIFO (Last In, First Out).        |
| 3     | Fila (Queue)            | Estrutura de dados que segue o princípio FIFO (First In, First Out).       |
| 4     | Lista Encadeada         | Estrutura de dados onde cada elemento aponta para o próximo.                |
| 5     | Dicionário (Dictionary) | Estrutura que armazena pares de chave-valor.                                |
| 6     | Hash Table              | Estrutura que implementa um mapeamento eficiente de chaves para valores.    |
| 7     | Grafo (Graph)           | Estrutura composta por vértices e arestas, usada para modelar relacionamentos. |
| 8     | Matriz (Matrix)         | Estrutura bidimensional de dados, representada em linhas e colunas.         |
| 9     | Vetor (Vector)          | Tipo de array de uma dimensão em que os elementos estão em sequência.       |
| 10    | Heap                    | Estrutura de dados usada para criar uma fila de prioridade.                 |

## 3. Conceitos de POO (Programação Orientada a Objetos)

| Ordem | Termo                   | Descrição                                                                 |
|-------|-------------------------|---------------------------------------------------------------------------|
| 1     | Classe                  | Modelo ou blueprint que define um objeto.                                 |
| 2     | Objeto                  | Instância de uma classe que contém atributos e métodos.                    |
| 3     | Herança                 | Mecanismo que permite que uma classe derive características de outra.      |
| 4     | Polimorfismo            | Capacidade de um método de se comportar de diferentes maneiras com base em sua classe. |
| 5     | Encapsulamento          | Ocultação dos detalhes internos de uma classe e exposição de uma interface pública. |
| 6     | Abstração               | Processo de ocultar detalhes complexos e mostrar apenas o essencial.       |
| 7     | Interface               | Contrato que define métodos que uma classe deve implementar.               |
| 8     | Sobrecarga (Overloading) | Definição de vários métodos com o mesmo nome, mas parâmetros diferentes.   |
| 9     | Construtor              | Método especial usado para inicializar um objeto de uma classe.            |
| 10    | Destrutor               | Método chamado quando um objeto é destruído para liberar recursos.          |

## 4. Paradigmas de Programação

| Ordem | Termo                   | Descrição                                                                 |
|-------|-------------------------|---------------------------------------------------------------------------|
| 1     | Programação Funcional   | Paradigma que trata a computação como a avaliação de funções matemáticas.  |
| 2     | Programação Procedural  | Paradigma que se baseia em procedimentos ou rotinas.                       |
| 3     | Programação Orientada a Objetos (POO) | Paradigma baseado em objetos que interagem entre si.                     |
| 4     | Programação Declarativa | Paradigma onde o foco é o "o que fazer" e não "como fazer".               |
| 5     | Programação Imperativa  | Paradigma que descreve como um programa opera por meio de instruções explícitas. |
| 6     | Programação Concorrente | Paradigma onde várias operações são executadas simultaneamente.           |
| 7     | Programação Assíncrona  | Paradigma que permite que tarefas sejam realizadas sem bloquear a execução do programa. |
| 8     | Programação Reativa     | Paradigma orientado por eventos e fluxos de dados dinâmicos.               |

## 5. Banco de Dados

| Ordem | Termo                   | Descrição                                                                 |
|-------|-------------------------|---------------------------------------------------------------------------|
| 1     | SQL (Structured Query Language) | Linguagem usada para gerenciar e consultar dados em bancos de dados relacionais. |
| 2     | NoSQL                   | Categoria de bancos de dados que não utilizam tabelas relacionais.        |
| 3     | Índice (Index)          | Estrutura que melhora a velocidade das consultas de um banco de dados.     |
| 4     | Chave Primária          | Campo ou conjunto de campos que identificam de forma única um registro em uma tabela. |
| 5     | Chave Estrangeira       | Campo em uma tabela que referencia a chave primária de outra tabela.       |
| 6     | Transação               | Conjunto de operações que devem ser executadas como uma unidade única.    |
| 7     | Normalização            | Processo de organização de dados em um banco para reduzir redundâncias.    |
| 8     | Desnormalização         | Processo de adicionar redundâncias a um banco de dados para melhorar a performance. |
| 9     | Stored Procedure        | Conjunto de instruções SQL pré-compiladas que podem ser executadas no banco de dados. |
| 10    | Trigger                 | Procedimento que é automaticamente executado em resposta a certos eventos em uma tabela. |

## 6. Segurança

| Ordem | Termo                   | Descrição                                                                 |
|-------|-------------------------|---------------------------------------------------------------------------|
| 1     | Autenticação            | Processo de verificar a identidade de um usuário ou sistema.              |
| 2     | Autorização             | Processo de conceder ou negar permissão para acessar um recurso.           |
| 3     | Criptografia            | Técnica de proteger dados convertendo-os em uma forma ilegível sem uma chave específica. |
| 4     | Firewall                | Sistema de segurança que monitora e controla o tráfego de rede com base em regras de segurança. |
| 5     | XSS (Cross-Site Scripting) | Ataque em que scripts maliciosos são injetados em sites confiáveis.        |
| 6     | SQL Injection           | Tipo de ataque onde um invasor insere código SQL malicioso em uma consulta. |
| 7     | Token de Segurança      | Código de acesso temporário que autentica solicitações de um usuário.      |
| 8     | HTTPS                   | Versão segura do HTTP que criptografa os dados entre o cliente e o servidor. |
| 9     | VPN (Virtual Private Network) | Rede privada que permite que dados sejam transmitidos de forma segura pela internet. |
| 10    | Ataque de DDoS (Distributed Denial of Service) | Ataque em que múltiplos sistemas comprometidos inundam um alvo com tráfego. |

## 12. Padrões de Design

| Ordem | Termo                   | Descrição                                                                 |

| 3     | Observer                | Padrão onde objetos notificam outros objetos sobre mudanças em seu estado. |
| 4     | Strategy                | Padrão que permite selecionar algoritmos em tempo de execução.            |
| 5     | Decorator               | Padrão que permite adicionar funcionalidades a um objeto de forma dinâmica. |
| 6     | Adapter                 | Padrão que permite que classes com interfaces incompatíveis trabalhem juntas. |
| 7     | Facade                  | Padrão que fornece uma interface simplificada para um sistema complexo.   |
| 8     | Proxy                   | Padrão que fornece um substituto ou ponto de controle para acessar outro objeto. |
| 9     | Command                 | Padrão que encapsula uma solicitação como um objeto, permitindo parametrização e enfileiramento de requisições. |
| 10    | Chain of Responsibility  | Padrão que permite passar solicitações por uma corrente de manipuladores. |

## 16. Qualidade de Código

| Ordem | Termo                   | Descrição                                                                 |
|-------|-------------------------|---------------------------------------------------------------------------|
| 1     | Code Smell              | Indício de possíveis problemas no código que podem afetar sua manutenção ou performance. |
| 2     | Refatoração             | Processo de modificar o código sem alterar seu comportamento externo, com o objetivo de melhorar sua estrutura interna. |
| 3     | Linting                 | Processo de analisar código-fonte em busca de erros, más práticas ou violações de estilo. |
| 4     | Clean Code              | Código que segue boas práticas, é legível e fácil de entender e manter.   |
| 5     | Technical Debt          | Compromissos tomados para ganhar tempo no desenvolvimento que precisam ser resolvidos posteriormente para evitar problemas no futuro. |
| 6     | Code Coverage           | Medida que indica a porcentagem do código que é coberta por testes.       |
| 7     | Continuous Refactoring  | Prática de melhorar continuamente o código durante o desenvolvimento, ao invés de realizar grandes refatorações periodicamente. |
| 8     | Pair Programming        | Técnica em que dois programadores trabalham juntos no mesmo código, onde um escreve enquanto o outro revisa. |
| 9     | Code Review             | Processo de revisão do código por outros desenvolvedores para encontrar erros e melhorar a qualidade. |
| 10    | Coding Standards        | Conjunto de regras e diretrizes que os desenvolvedores seguem para escrever código consistente e de alta qualidade. |
